# AI 助手技能（Skills）

AI 助手支持「技能」机制：一个技能是一段附加给 AI 的**指令模板**，在用户提问时被自动或手动激活，让 AI 按照预设的结构、步骤和输出格式来回答。

技能以 **Markdown 文件**（带 YAML frontmatter）的形式存储在应用数据目录的 `skills/` 下，用户可以查看、编辑、删除，也可以新建自己的技能。

> **技能 vs 工具**：技能约束 AI「怎么说」（回答结构/格式），工具让 AI「能查到」（获取实时行情、大盘指数等数据）。例如问「今天大盘怎么样」需要的是工具而非技能。工具机制是应用内置的，详见 [AI 助手工具（Tools）](./ai-tools.md)。

## 技能文件格式

每个技能是一个 `.md` 文件，文件名（去后缀）即技能的 `id`，内容包含一个 frontmatter 元信息块和一段 Markdown 正文：

```markdown
---
name: 持仓风险分析
description: 分析当前投资组合的风险敞口、集中度与最大回撤
trigger: 风险,集中度,回撤,敞口
enabled: true
---

# 持仓风险分析

## 你的任务
基于已注入的「当前投资组合快照」，对用户的组合做一次完整的风险体检。

## 分析步骤
1. 按市值计算每只持仓的占比……
2. ……

## 输出格式
使用 Markdown 表格 + 「⚠️ 风险点」列表。
```

### Frontmatter 字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `name` | 是 | 人类可读的技能名称（可中文），显示在列表与徽标中。 |
| `description` | 是 | 一句话描述，显示在技能列表与 `/` 选择列表中。 |
| `trigger` | 否 | 逗号分隔的关键词。当用户**最新一条消息**包含任一关键词时该技能自动激活（大小写不敏感）。 |
| `enabled` | 否 | `true` / `false`，默认 `true`。关闭后不参与自动激活，但仍可被 `/` 手动调用。 |

### 正文

正文是技能的核心：当技能激活时，整段正文会被作为一条额外的 `system` 消息追加到对话中，告诉 AI 应该如何回答。建议结构：

- `# 技能标题` — 一级标题
- `## 你的任务` — 简述这次分析的目标
- `## 分析步骤` — 编号列表给出执行步骤
- `## 输出格式` — 约定 Markdown 结构（表格、列表、标题等）
- `## 注意` — 限制与免责声明

正文支持完整的 Markdown 语法（标题、列表、表格、代码块等），但**不会**被执行为脚本——它只是给 AI 的自然语言指令。

## 激活方式

技能有两种激活途径：

1. **自动激活** — 当用户开启 AI 助手中的「注入数据」开关、且最新一条消息包含技能的某个 `trigger` 关键词时，该技能自动激活。同一回合可以激活多个技能。
2. **手动激活** — 在 AI 助手输入框中输入 `/`，会弹出技能选择列表（支持按名称/id 过滤），选中后该技能在**下一次发送**时生效（仅一次）。也可点击欢迎页的「快捷技能」标签直接用预设开场白发起对话。

### `/` 弹窗的键盘操作

| 按键 | 作用 |
| --- | --- |
| `↑` / `↓` | 在候选技能之间移动高亮（支持首尾循环） |
| `Enter` | 选中当前高亮的技能（不发送消息） |
| `Esc` | 关闭弹窗（移除输入框尾部的 `/…`） |
| `Ctrl/⌘+Enter` | 仍照常发送消息 |

### 待激活可视化

通过 `/` 选中的技能会立刻在输入框上方显示为紫色「待激活」标签，可点击标签上的 `×` 取消单个技能。多次 `/` 选择会叠加（同一技能自动去重）。发送后这些标签消失，AI 回复气泡顶部会显示紫色徽标 `⚡ 已用技能：{技能名称}`。

### 关键词匹配规则

为避免误激活，触发词匹配按以下规则：

- **CJK 关键词**（含中文/日文等非 ASCII 字符）：子串匹配。CJK 没有词边界，"夏普" 命中 "夏普比率" 是预期行为。
- **ASCII 关键词**（纯英文/数字）：整词匹配。`risk` 不会命中 `riskier`、`assessing` 等包含该子串的更长单词。

手动激活优先于自动激活：一旦显式选择了技能，后端不会再做关键词匹配。

## 内置技能

应用首次启动时会自动把以下 10 个内置技能「落盘」到用户技能目录，用户可以直接编辑或删除它们：

| id | 名称 | 用途 |
| --- | --- | --- |
| `portfolio-risk-analysis` | 持仓风险分析 | 集中度、回撤、波动率体检 |
| `trade-review` | 交易决策复盘 | 点评近期买卖决策的得失 |
| `quarterly-report` | 季度回顾报告 | 生成结构化的季度回顾 Markdown |
| `performance-diagnosis` | 绩效指标诊断 | 解读收益、夏普、最大回撤等指标 |
| `market-pulse` | 今日市场扫描 | 今日大盘指数 + 持仓当日表现 |
| `stock-deep-dive` | 个股深度诊断 | 指定个股的行情、走势、持仓盈亏 |
| `return-attribution` | 收益归因分析 | 盈亏按市场/类别/个股/月份拆解 |
| `dividend-income` | 分红收入分析 | 分红/利息收入汇总与收益率估算 |
| `allocation-checkup` | 资产配置体检 | 市场分布、个股集中度、风格配置 |
| `options-review` | 期权策略复盘 | 期权持仓、到期风险、权利金统计 |

> **编辑内置技能**：编辑后的内置技能会变为「自定义」来源（`source: user`），原内置版本可通过「恢复内置」按钮重置回来。
> **删除内置技能**：删除只是移除落盘的 `.md` 文件，「恢复内置」会重新生成。

## 管理入口

打开 **设置 → AI 配置 → 技能管理** 面板可以：

- 查看所有技能（含内置与自定义）、其触发词与启用状态
- 一键启用 / 停用某个技能
- 编辑（内置技能会另存为自定义副本）。编辑弹窗右侧支持「编辑 / 预览」切换，预览模式实时渲染 Markdown。
- **克隆**：把任一技能复制为新 id（自动加 `-copy` 后缀），适合基于内置技能派生自定义版本
- **测试**：跳转到 AI 助手并预激活该技能，输入问题后发送即生效
- **导出**：把技能另存为 `.md` 文件（系统保存对话框选择路径）
- **导入**：从 `.md` 文件加载技能（id 冲突时自动加 `-2`/`-3` 后缀）
- 删除技能
- 新建技能（弹窗表单：ID、名称、描述、触发词、正文）
- 「恢复内置」：清除对内置技能的改动并重新写入出厂版本

## 文件位置

技能文件存储在应用数据目录的 `skills/` 子目录下：

- macOS: `~/Library/Application Support/com.stock-portfolio-manager.app/skills/`
- Windows: `%APPDATA%\com.stock-portfolio-manager.app\skills\`
- Linux: `~/.local/share/com.stock-portfolio-manager.app/skills/`

> 应用启动时若 `skills/` 目录不存在会自动创建，并补齐缺失的内置技能文件。用户手动放入符合格式的 `.md` 文件也会被自动识别（下次刷新技能列表时生效）；也可在管理面板点「导入」按钮从任意路径加载。

## 编写示例

一个最小可用的自定义技能：

```markdown
---
name: 现金流分析
description: 分析组合的现金占比与近 30 天净流入流出
trigger: 现金,现金流,流入,流出
enabled: true
---

# 现金流分析

## 任务
基于持仓快照中的「现金类」持仓和近期交易，评估组合的流动性。

## 输出
1. 当前现金占比（现金市值 / 总市值）
2. 近 30 天净现金流（BUY 为流出，SELL/PAY 为流入）
3. 一句话建议：现金是否充足
```

## 注意事项

- 技能正文只是给 AI 的提示，**不能保证** AI 严格遵循——不同模型遵循指令的能力有差异。
- 技能激活依赖「注入数据」提供持仓快照；关闭该开关后，技能仍会激活但 AI 可能缺少数据。
- `trigger` 关键词匹配是子串包含（大小写不敏感），过短的词（如单字「涨」）可能误触发，建议使用 2 字及以上的具体词。
- 所有技能共享同一份 system prompt 预算；同时激活过多技能会挤占上下文，建议一次激活 1–3 个。
