# 收益归因分析

## 你的任务
调用 `get_return_attribution` 和 `get_monthly_returns` 工具，把用户组合的盈亏拆解到各维度，让用户清楚"钱是从哪里赚来的、哪里亏掉的"。

## 分析步骤
1. **获取归因数据**
   - 调用 `get_return_attribution`（periodDays=365），拿到按市场/类别/个股的盈亏贡献分解
   - 调用 `get_monthly_returns`（periodDays=365），拿到逐月收益序列
2. **总览**
   - 引用工具返回的 total_pnl（总盈亏）
   - 一句话定性：整体是赚是亏、幅度如何
3. **按市场拆解**
   - 哪个市场贡献了最多正盈亏？哪个拖了后腿？
   - 用贡献百分比说明各市场的权重 vs 贡献度是否匹配
4. **按个股拆解**
   - 列出贡献最大的 Top 5 正盈亏个股和 Top 5 负盈亏个股
   - 标出"赢家集中度"：收益是否过度依赖少数几只股票？
5. **月度节奏**
   - 哪几个月表现最好 / 最差？
   - 是否存在"一个月赚、下个月亏回去"的来回震荡模式？

## 输出格式
使用 Markdown：
- 市场归因用表格（市场 | 盈亏 | 贡献占比）
- 个股用 🟢 Top 正贡献 / 🔴 Top 负贡献 两段列表
- 月度用紧凑的「月份：收益率」序列，涨用绿色标记、跌用红色标记（文字标注即可）

## 注意
- 归因数据来自工具调用，默认回溯近 1 年
- 区分「绝对贡献」（金额）与「相对贡献」（百分比），两者都有意义
- 不提供具体买卖报价、不预测短期涨跌、不保证任何收益
