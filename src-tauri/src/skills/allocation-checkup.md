# 资产配置体检

## 你的任务
调用 `get_dashboard_summary` 和 `get_holdings_detail` 工具，全面检查用户组合的资产配置结构，指出不合理之处。

## 分析步骤
1. **获取配置数据**
   - 调用 `get_dashboard_summary`，拿到总市值、总成本、总盈亏、按市场拆分
   - 调用 `get_holdings_detail`，拿到逐只持仓明细（含市值、占比可算）
2. **总览**
   - 总资产规模（USD）、累计盈亏金额与百分比
   - 一句话定性当前组合体量
3. **市场分布**
   - 美股 / 港股 / A股各占比多少？
   - 是否过度集中于单一市场？（单一市场 > 70% 需提醒）
   - 跨市场对冲是否充分？
4. **个股集中度**
   - 计算每只持仓占总市值的百分比
   - 标出重仓股（占比 ≥ 15%）
   - 计算前 5 大持仓的合计占比（CR5），> 60% 说明集中度较高
5. **风格与类别**
   - 按类别（现金类 / 分红股 / 成长股 / 套利）汇总
   - 现金占比是否过高（> 30%）或过低（< 5%）？
6. **配置建议**
   - 基于上述分析，给出 2–3 条客观的配置优化方向

## 输出格式
使用 Markdown：
- 市场分布用饼图风格的表格或文字百分比
- 个股集中度用表格（标的 | 市值 | 占比）标红重仓项
- 建议用 ⚠️ 标记需关注项、✅ 标记合理项

## 注意
- 市值数据来自工具调用（缓存行情），可能略有延迟
- "合理"的配置因人而异，给出的建议是基于一般原则的参考，非绝对标准
- 不提供具体买卖报价、不预测短期涨跌、不保证任何收益
