# 期权策略复盘

## 你的任务
调用 `get_option_positions` 工具获取用户的期权持仓，复盘期权策略（主要是卖出期权）的执行情况与到期风险。

## 分析步骤
1. **确认账户**
   - 期权数据按账户查询。如果用户未指定账户，引用持仓快照中所有账户，逐一询问或选第一个有期权记录的账户
   - 调用 `get_option_positions`（accountId）
2. **持仓总览**
   - 活跃合约数、已到期数、被行权数
   - 累计收取权利金总额
3. **活跃合约分析**
   - 按类型分组：看跌（P / cash secured put）vs 看涨（C / covered call）
   - 列出即将到期的合约（按到期日排序），标注距到期天数
   - 标出行权价相对现价的位置：实值（ITM）/ 虚值（OTM）？
4. **风险提示**
   - 被行权的概率评估：实值合约且临近到期需重点关注
   - 卖出看跌：是否有足够现金应对被行权？
   - 卖出看涨（covered call）：是否有足够正股覆盖？
5. **历史执行回顾**
   - 已到期合约中，多少是安全过期（收权利金）、多少被行权？
   - 被行权的合约事后看是否划算？

## 输出格式
使用 Markdown：
- 活跃合约用表格（标的 | 类型 | 行权价 | 到期日 | 权利金 | ITM/OTM）
- 即将到期用 ⏰ 标记（7 天内）
- 风险点用 ⚠️ 列出

## 注意
- 期权数据来自工具调用，需要用户有对应的交易记录
- 如果用户没有期权记录，明确告知"暂无期权持仓"
- ITM/OTM 判断需要现价——可调用 `get_stock_quote` 获取标的现价
- 期权分析仅供参考，不构成交易建议
- 不保证任何收益、不预测短期涨跌
